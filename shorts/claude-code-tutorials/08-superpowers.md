# VIBE-CODING BEZ KODU #8: SUPERPOWERS

Platform: TikTok/Reels/Shorts (agnostic) | Length: 60s | Type: Educational
Seria: Porady do vibe-codingu dla osób nietechnicznych

## Hook Options

1. **Proof/Scale:** „Jeden plugin. Sto trzydzieści siedem tysięcy gwiazdek."
2. **Contrarian:** „To nie jest plugin. To system operacyjny dla Claude'a."
3. **Question:** „Dlaczego jeden plugin ma więcej gwiazdek niż cały GitHub Copilot?"

Rekomendacja: **Hook 1** — konkretna liczba w pierwszej sekundzie budzi ciekawość. 7 słów. Skala 137K jest tak absurdalna, że wymusza scroll-stop. Hook 2 jako backup — reframe jest mocny, ale wymaga kontekstu.

## Script (Hook 1)

[0-2s] „Jeden plugin. Sto trzydzieści siedem tysięcy gwiazdek."
  📷 Shot: Talking head, bliski kadr, pauza po „plugin"
  📝 Text: 1 PLUGIN → 137K ⭐
  📷 Overlay: GitHub stars counter animacja
  🎵 Audio: Single deep bass hit na „gwiazdek"
  ⏱️ Pacing: Powolny, dramatyczny, dwa zdania = dwa uderzenia

[2-8s] „Superpowers od obra. I nie — to nie jest kolejny skill. To jest metodologia. Zmienia JAK Claude myśli."
  📷 Shot: Screen recording — repo Superpowers na GitHubie, scroll przez README
  📝 Text: SUPERPOWERS by obra — METODOLOGIA
  🎵 Audio: Lo-fi beat wchodzi, lekki build-up
  ⏱️ Pacing: Szybki, konwersacyjny, nacisk na „JAK"

[8-18s] „Numer jeden — Socratic Brainstorming. Zamiast od razu kodować, Claude najpierw zadaje ci pytania. Dopytuje. Rozumie problem, zanim napisze linijkę."
  📷 Shot: Split screen — prompt → Claude zadaje 3 pytania zamiast odpowiedzi
  📝 Text: 1. SOCRATIC BRAINSTORMING — myśli, potem działa
  🎵 Audio: Notification ding na każde pytanie
  ⏱️ Pacing: Średni, budujący zrozumienie

[18-28s] „Dwa — Test-Driven Development. Claude pisze test PRZED kodem. A potem koduje, dopóki test nie przejdzie. Nie zgaduje. Wie, kiedy skończył."
  📷 Shot: Screen recording — terminal: testy fail (czerwone) → pass (zielone)
  📝 Text: 2. TDD — test → kod → pass ✅
  🎵 Audio: Error buzz (fail) → success chime (pass)
  ⏱️ Pacing: Budujący — fail → pass = satysfakcja

[28-38s] „Trzy — Sub-agenty. Claude odpala osobne wątki, które pracują równolegle. Jeden pisze frontend, drugi backend, trzeci testy. Jednocześnie."
  📷 Shot: Animacja — jeden agent rozdziela się na trzy, każdy w swoim oknie
  📝 Text: 3. SUB-AGENTY — ×3 równolegle
  🎵 Audio: Multiplication SFX, trzy dźwięki narastające
  ⏱️ Pacing: Przyspieszający, energia

[38-45s] „ALE — to nie są trzy oddzielne narzędzia. To JEDEN system. I w tym jest klucz."
  📷 Shot: Talking head, lean-in, bezpośredni kontakt wzrokowy, gest łączenia dłoni
  📝 Text: 3 NARZĘDZIA? → 1 SYSTEM.
  🎵 Audio: Drop do ciszy na „klucz"
  ⏱️ Pacing: Wolny, dramatyczna pauza

[45-55s] „Jeden skill to apka. System skilli to system operacyjny. Superpowers nie daje Claude'owi narzędzi — daje mu instynkty. Zawsze planuj. Zawsze testuj. Zawsze dziel pracę."
  📷 Shot: Talking head → overlay: ikony apek vs ikona OS. Potem trzy hasła jedno po drugim
  📝 Text: APKA vs SYSTEM OPERACYJNY → Planuj. Testuj. Dziel.
  🎵 Audio: Build-up, trzy uderzenia na trzy hasła
  ⏱️ Pacing: Przyspieszający ku punchline, trzy hasła = staccato

[55-60s] „A w następnym odcinku — co się stanie, kiedy każdy zacznie tworzyć takie systemy. Skills economy."
  📷 Shot: Talking head + overlay „Cz. 9: SKILLS ECONOMY"
  📝 Text: #9 → SKILLS ECONOMY
  🎵 Audio: Outro sting, open loop tension
  ⏱️ Pacing: Szybki, niedokończony, open loop

## But/Therefore Test ✓

> „Jeden plugin. 137 tysięcy gwiazdek." (HOOK — skala)
> THEREFORE → „Superpowers od obra. To metodologia." (credibility)
> THEREFORE → „Numer jeden — Socratic Brainstorming, Claude pyta zamiast zgadywać."
> THEREFORE → „Dwa — TDD, test przed kodem, wie kiedy skończył."
> THEREFORE → „Trzy — Sub-agenty, trzy wątki równolegle."
> BUT → „To nie są trzy narzędzia. To jeden system."
> THEREFORE → „Skill to apka. System skilli to OS. Superpowers daje instynkty."
> THEREFORE → „Planuj. Testuj. Dziel." (PUNCHLINE → loops to „Jeden plugin" — jeden system, trzy instynkty)

## Production Notes

- **Key visual moments:** (1) GitHub stars counter 137K — wow/scale, (2) Split screen Socratic — Claude pyta zamiast odpowiada, (3) Terminal red→green TDD, (4) Animacja rozdzielenia na 3 sub-agenty, (5) Staccato „Planuj. Testuj. Dziel." z trzema uderzeniami
- **Recommended audio/music mood:** Lo-fi tech beat, budujący. Deep bass na hook. Cisza na reframe (38-45s). Trzy uderzenia na punchline. Build tension do open loop.
- **Loop point:** „Planuj. Testuj. Dziel." (55s) → „Jeden plugin." (0s). Punchline mówi CO daje system — hook mówi JAKI to system. Widz chce zobaczyć ponownie.
- **CTA type:** Level 5 — open loop na „skills economy" = makro-trend, budzi ciekawość. Logiczny next step: poznałeś najlepszy system → co jeśli KAŻDY tworzy systemy?
- **Dodatkowe:** Rozważyć B-roll z prawdziwym terminalem i Superpowers w akcji. Instalacja: `/install-github obra/superpowers` — pokazać na ekranie jako bonus wizualny.

## Content Lego

**Case study pojedynczego narzędzia** — głęboki dive w jeden plugin zamiast listy. Testujemy, czy „1 tool deep" daje lepszą retention niż „5 tools wide" (#2). Jeśli >45% retention w 30s, format „anatomy of [tool]" można powtórzyć dla Context7, Stitch, itp. Testujemy też **analogię jako reframe** (apka vs OS) — czy punchline z porównaniem buduje shareability.

## STEPPS Score: 5/6

- ✅ **Social Currency** — widz wie DLACZEGO Superpowers ma 137K gwiazdek, gdy inni mówią tylko „zainstaluj"
- ✅ **Trigger** — za każdym razem, gdy Claude zaczyna kodować bez pytania / bez planu / bez testów
- ✅ **Emotion** — awe (high arousal) — skala 137K + reframe „instynkty" a nie narzędzia
- ❌ **Public** — format standardowy, trudno zobaczyć kto „używa Superpowers"
- ✅ **Practical Value** — 3 konkretne features do zapamiętania, jedno polecenie instalacji
- ✅ **Story** — narracja od skali (137K) przez trzy features do reframe (apka→OS) — łuk dramatyczny
