# VIBE-CODING BEZ KODU #3: ZRÓB WŁASNY SKILL

Platform: TikTok/Reels/Shorts (agnostic) | Length: 60s | Type: Educational
Seria: Porady do vibe-codingu dla osób nietechnicznych

## Hook Options

1. **Proof-first:** „Zrobiłem własnego skilla w 3 minuty. Bez kodu."
2. **Contrarian:** „Własny skill to zwykły plik tekstowy. Serio."
3. **Question:** „Co gdybyś mógł nauczyć Claude nowej komendy w 3 minuty?"

Rekomendacja: **Hook 1** — zapowiedziany w CTA odcinka #2 ("zbudować własny skill od zera. Żadnego kodu."). Proof + time constraint + contrarian twist ("bez kodu"). Front-loaded value: CO (skill) + JAK SZYBKO (3 min) + ZASKOCZENIE (bez kodu).

## Script (Hook 1)

[0-2s] „Zrobiłem własnego skilla w trzy minuty. Bez kodu."
  📷 Shot: Talking head, bliski kadr, lekki uśmiech na „bez kodu"
  📝 Text: WŁASNY SKILL → 3 MIN
  🎵 Audio: Whoosh na „bez kodu"
  ⏱️ Pacing: Szybki, pewny, pauza po „minuty"

[2-7s] „W poprzednim odcinku zainstalowałeś pięć skilli. Ale co jeśli potrzebujesz czegoś, czego nikt nie zrobił?"
  📷 Shot: Medium shot → wstawka: szybki flash 5 skilli z #2 (overlay)
  📝 Text: 5 skilli ✓ → ale brakuje jednego?
  🎵 Audio: Notification ding na flash
  ⏱️ Pacing: Średni, pytanie retoryczne zawieszone

[7-12s] „Więc robisz własny. I to nie wymaga żadnego programowania — bo skill to dosłownie plik tekstowy."
  📷 Shot: Talking head lean-in → screen: Finder/explorer z plikiem SKILL.md
  📝 Text: SKILL = plik tekstowy
  🎵 Audio: Record scratch na „plik tekstowy"
  ⏱️ Pacing: Szybki, demistyfikacja — kluczowy moment „aha"

[12-18s] „Pokażę ci. Chcę skilla, który podsumowuje dowolny plik w trzech punktach. Robię folder — .claude, skills, podsumuj."
  📷 Shot: Screen recording — terminal: mkdir -p .claude/skills/podsumuj
  📝 Text: .claude/skills/podsumuj/
  🎵 Audio: Typing SFX, folder creation sound
  ⏱️ Pacing: Szybki, hands-on energy

[18-25s] „W środku tworzę plik SKILL.md. Na górze? Trzy linijki: name, description — po angielsku, bo tak chce Claude. To cały config."
  📷 Shot: Screen recording — edytor, wpisywanie YAML frontmatter: name: podsumuj, description: Summarize any file in 3 bullet points
  📝 Text: 3 linijki = cały config
  🎵 Audio: Subtle keystroke SFX
  ⏱️ Pacing: Średni, krok-po-kroku

[25-32s] „A pod spodem? Piszesz instrukcje. Normalnym językiem. Po polsku, po angielsku, jak chcesz. Ja piszę: przeczytaj plik, wyciągnij trzy najważniejsze rzeczy, wypisz jako bullet pointy."
  📷 Shot: Screen recording — pisanie markdown pod frontmatter, zoom na tekst instrukcji
  📝 Text: Instrukcje = normalny język
  🎵 Audio: Build-up, pen-on-paper SFX
  ⏱️ Pacing: Średni → przyspieszenie na wyliczankę

[32-38s] „Zapisuję. I teraz patrzcie — wpisuję /podsumuj, podaję plik i… boom."
  📷 Shot: Screen recording — terminal: /podsumuj → Claude przetwarza → output 3 bullet pointy
  📝 Text: /podsumuj → gotowe ✓
  🎵 Audio: Satisfying click na „boom", notification ding na output
  ⏱️ Pacing: Szybki, reveal moment — pauza przed „boom"

[38-44s] „ALE to nie jest zwykły prompt. Różnica? Prompt musisz wklejać za każdym razem. Skill żyje w projekcie — działa zawsze, dla każdego kto otworzy ten folder."
  📷 Shot: Talking head, gest porównania (lewa ręka = prompt, prawa = skill)
  📝 Text: Prompt = jednorazowy → Skill = permanentny
  🎵 Audio: Drop na „zawsze"
  ⏱️ Pacing: Wolniejszy, kontrast — kluczowe rozróżnienie

[44-50s] „Cały skill to dziesięć linijek. Żadnego kodu, żadnej konfiguracji, żadnego terminala. Tylko plik z instrukcjami."
  📷 Shot: Screen recording — pełny SKILL.md widoczny, zoom out żeby pokazać prostotę
  📝 Text: 10 linijek. To wszystko.
  🎵 Audio: Cisza → single bass hit na „to wszystko"
  ⏱️ Pacing: Pauza. Powoli. Punchline.

[50-55s] „Trzy minuty temu nie miałeś skilla. Teraz masz. Bez jednej linijki kodu."
  📷 Shot: Close-up, lean-in, bezpośredni kontakt wzrokowy
  📝 Text: 0 → 1 w 3 minuty
  🎵 Audio: Subtle build
  ⏱️ Pacing: Powolny, loopuje z hookiem

[55-60s] „A w następnym odcinku pokażę ci skills, które nie mają nic wspólnego z kodem. Spoiler: tam się dzieje magia."
  📷 Shot: Talking head + overlay „Cz. 4: Skills nie-do-kodu"
  📝 Text: SKILLS ≠ KOD → Część 4
  🎵 Audio: Outro sting, mystery tone
  ⏱️ Pacing: Szybki, open loop z intrygą

## But/Therefore Test ✓

> „Zrobiłem własnego skilla w 3 minuty." (HOOK)
> BUT → „Co jeśli potrzebujesz czegoś, czego nikt nie zrobił?"
> THEREFORE → „Robisz własny — i to jest plik tekstowy."
> THEREFORE → „Pokażę ci: folder, SKILL.md, trzy linijki config."
> THEREFORE → „Instrukcje piszesz normalnym językiem."
> THEREFORE → „Wpisuję /podsumuj — boom, działa."
> BUT → „To nie jest zwykły prompt — skill żyje w projekcie, działa zawsze."
> THEREFORE → „Cały skill to 10 linijek. Żadnego kodu."
> THEREFORE → „3 minuty temu nie miałeś skilla. Teraz masz." (PUNCHLINE → loops to HOOK)

## Production Notes

- **Key visual moments:** (1) Record scratch na „plik tekstowy" — moment demistyfikacji, (2) Screen recording tworzenia folderu + SKILL.md — proof it's real, (3) Reveal: /podsumuj → output — satisfying payoff, (4) Zoom out na cały SKILL.md — „to tyle?" moment
- **Recommended audio/music mood:** Lo-fi chill beat z wyraźnym tempem. Record scratch w 7-12s. Typing/creation SFX w środkowej sekcji (12-32s). Cisza + bass hit na punchline (44-50s). Mystery tone na CTA.
- **Loop point:** „Trzy minuty temu nie miałeś skilla. Teraz masz. Bez jednej linijki kodu." (50-55s) → „Zrobiłem własnego skilla w trzy minuty. Bez kodu." (0s). Punchline jest parafrazą hooka.
- **CTA type:** Level 5 — open loop na „skills nie-do-kodu" = intrygujący pivot. Logiczny next step po „zbuduj własny" = „do czego jeszcze możesz to użyć?"

## Content Lego Being Tested

**Live walkthrough (build X in Y minutes)** — format „zrób to ze mną" z timer pressure (3 min). Testujemy czy screen recording krok-po-kroku utrzymuje retention lepiej niż talking head z wstawkami (#1) i listicle (#2). Jeśli completion rate >45%, format „zbuduj X w Y minut" wchodzi do rotacji.

## STEPPS Score: 4/6

- ✅ **Social Currency** — „umiem budować własne skills" = insider status, wyżej niż „umiem instalować cudze"
- ✅ **Trigger** — za każdym razem gdy brakuje skilla / powtarzasz ten sam prompt
- ✅ **Emotion** — empowerment (high arousal) — „to AŻ TAKIE proste?" + satisfaction z zrobienia czegoś samemu
- ❌ **Public** — screen recording, ale brak „widocznego" elementu do naśladowania w real life
- ✅ **Practical Value** — konkretny tutorial: folder → plik → 3 linijki → działa
- ❌ **Story** — tutorial format, nie narracja (ale progress arc: 0 → 1 skill daje mini-story feel)
