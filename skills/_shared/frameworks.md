# Structural Frameworks Reference

## Kallaway Storytelling Playbook

### 1. The Dance — But/Therefore (South Park Method)
Conflict creates open loops, context closes them. Alternating CONTEXT <-> CONFLICT maintains attention.

- Arrange story beats so BUT/THEREFORE fits between them, never AND THEN
- In practice: Statement A... THEREFORE B... BUT C... THEREFORE D...
- Weave 2-4 mini-conflicts (head-fakes) in first 30 seconds

**Template:** This happened, THEREFORE ____. BUT ____, THEREFORE ____. BUT ____, THEREFORE ____.

### 2. Rhythm — Sentence Length Variation (Gary Provost)
Uniform sentence lengths bore and lower retention. Rhythm = musicality of text.

- Mix: very short sentences. Medium ones. Sometimes longer, building like a crescendo.
- Write each sentence on its own line — visually see the "jagged edge"
- In editing: mix shots/cuts according to sentence rhythm

### 3. Tone — Talk to One Friend
Conversational tone lowers "sales friction." Viewer feels a conversation, not a lecture.

- Write and record as if sending a voice note to a close friend
- Avoid "speaking TO" — switch to "talking WITH" (interjections, colloquial connectors, rhetorical questions)

**Template:** "Hey, quick thing. Ever seen ____? Look at this: _____. Sounds basic, but you'll see why it matters."

### 4. Direction — Start from the End (Last Dab)
Strong, loopable punchline increases watch-loop shares. The ending sets up the beginning.

- First write the last 1-2 sentences — make it "quote-worthy, forward-worthy"
- Then build beginning and middle to naturally deliver to the "last dab"
- In shorts: last line connects back to first (loop)

### 5. Story Lenses — Unique Angle (Category of One)
Topic alone isn't enough — you need a unique angle. The lens differentiates your content.

- List standard takes on the topic → cross them out. Find a rare angle
- Match lens to your credibility (experience, data, access)
- Test 3 lenses on the same topic — pick highest CTR potential

**Template:** Topic: ____. My lens: ____, because ____. Evidence/artifacts: ____.

### 6. Hook — Say It Clearly and Show It
Vision processes faster than hearing. Visual hook + clear thesis = fewer drop-offs.

- First line = story essence ("what this is about"), no cryptic intros
- Add immediate visual proof (object, chart, screen, effect). "Show while you tell"
- Make frame 0-2s readable in thumbnail and without audio

## Hormozi Script Formula
```
[0-2s]  BOLD CLAIM: "Here's how to [result] in [timeframe]"
[2-8s]  CREDIBILITY: "I did this with [proof]" / "This made me [result]"
[8-45s] FRAMEWORK: "Step 1... Step 2... Step 3..." (ultra-specific)
[45-55s] REFRAME: "Most people think [X], but actually [Y]"
[55-60s] CTA: Implied ("If you want more frameworks like this...")
```

Key rules:
- "Give away the secrets, sell the implementation"
- "One video, one idea, one CTA"
- "Volume negates luck" — 2-3 shorts DAILY to start

## Film Booth 4-Type Hook System

| Type | Mechanism | Template |
|------|-----------|----------|
| **Question** | Opens cognitive loop | "What happens when you [X]?" / "Why does nobody talk about [X]?" |
| **Statement** | Controversial/surprising claim | "Nobody needs [popular thing]." / "[X] is a complete lie." |
| **Situation** | Drops viewer into action | "So I'm standing in [place] and [unexpected thing happens]" |
| **Action** | Visual action without context | *[person does something strange]* → pure curiosity |

**Rule:** Never start with context. Context = permission to leave. Hook → then context.

## Paddy Galloway Curiosity Stack
Stack curiosity layers — more layers = stronger hook.

1. **Layer 1 — What:** What will/did happen
2. **Layer 2 — Stakes:** What's at stake (why it matters)
3. **Layer 3 — Tease:** Fragment of result without full answer

For short-form: 2 layers are enough. 3 = too long for <30s.

## PAS (Problem, Agitation, Solution)
From direct response copywriting (Gary Halbert, Dan Kennedy). Deliberately deepens pain before giving solution — increases perceived value.

1. **Problem:** Name the reader's problem in THEIR language
2. **Agitation:** Deepen the pain — consequences, worsening vision, social proof
3. **Solution:** Present solution as logical conclusion after Agitation

## PASTOR Framework (Ray Edwards)
Extended PAS with storytelling and social proof. Works for longer formats.

1. **P — Problem:** Identify and name the problem
2. **A — Amplify:** Strengthen consequences of not solving
3. **S — Story/Solution:** Tell story of someone who had same problem and solved it
4. **T — Transformation/Testimony:** Show result (before → after) + social proof
5. **O — Offer:** What you specifically propose
6. **R — Response:** CTA — what to do now

## Pixar Story Spine
```
Once upon a time, [context/status quo].
Every day, [routine].
One day, [inciting incident — something changed].
Because of that, [consequence 1].
Because of that, [consequence 2].
Because of that, [consequence 3 — critical point].
Until finally, [resolution/breakthrough].
And ever since that day, [new status quo — transformation].
The moral: [lesson].
```

## SCQA (McKinsey)
1. **Situation:** State of affairs the reader agrees with (common ground)
2. **Complication:** What changed / what went wrong / why status quo doesn't work
3. **Question:** Key question naturally arising from Complication
4. **Answer:** Your answer — solution, framework, proposal

## Content Legos (Kallaway)
Every video = modular building blocks. Improve one block at a time.

**7 Legos for Short Form:**
1. Topic
2. Angle
3. Hook format
4. Storytelling style
5. Visual format
6. Key visuals
7. Audio

For YouTube, add: Title + Thumbnail

**Method:** Change only ONE Lego per iteration to isolate what works.

## Speed to Value (Kallaway)
Don't share as much as possible — deliver ONE clear, valuable insight as FAST as possible.

- If value line comes after 7-10s (shorts) or 45-60s (YouTube) — cut the fluff above it
- Focus on speed of delivering value, not quantity of information
